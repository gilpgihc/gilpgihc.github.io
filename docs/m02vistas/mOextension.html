F. Vista secundaria

  • Cuando la barra de aplicación no está en la página principal, puede mostrar hasta 3 botones de ícono a la derecha. Deben ser las acciones más usadas.

  • Cuando hay más de 3 acciones, coloca las menos usadas en el overflow menu, que se despliega cliqueando el botón con el ícono ⋯ en iOS, y el ícono ⁝ en otros sistemas operativos.

  • Cuando en iOS no hay íconos de acción, el título debe ir centrado.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado top-app-bar para este tipo de páginas. Su comportamiento lo definimos con la clase TopAppBar.

  • Para el overflow menu, se define el elemento personalizado overflow-menu y su comportamiento lo definimos con la clase OverflowMenu.

  • Para el título, dentro de la la barra de aplicación se usa un elemento de tipo h1 con el atributo slot="headline". El texto debe ser corto y caber en un solo renglón, o de lo conrario se cortará.

  • Para mostrar un título más largo, hay que poner a la barra de aplicación la clase medium, agregar en esta barra, otro elemento de tipo h1 con el atributo slot="headline-small". Cuando la página tenga el scroll hasta arriba, se despliega este encabezado. Al hacer scroll, este título se compacta y se muestra el título definido con slot="headline".

  • El botón de navegación opcional, debe llevar al atributo slot="navigation".

  • Los botones de acción deben llevar al atributo slot="action".

  • El overflow menú debe llevar al atributo slot="overflow".

  • Para añadir elementos debajo del título, usa el atributo slot="adicional".

  • El botón de ícono para abrir el overflow menu, se coloca dentro de este menú usando el atributo slot="icon".

  • Los botones de acción se colocan dentro del overflow menu sin ningún atributo slot.
