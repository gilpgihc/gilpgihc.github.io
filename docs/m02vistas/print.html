B. Formulario para móvil

  • Los mensajes de error de de un componente, se muestran en otro componente conocido como texto de ayuda.

  • En este ejemplo, se muestra un texto de ayuda con el aributo id="msgGenero", colocado dentro de un elemento small

  • La forma usa el atributo novalidate.

  • El campo select usa el atributo oninput. para invocar la función copiaMensajes. y copiar los mensajes de error del campo de captura al texto de ayuda.

  • En el resto de la lección se muestra como dar apariencia móvil a la interfaz gráfica.

E. App básica

  • La distribución básica de las aplicaciónes móviles incluye una barra de aplicación que contiene el título de la aplicación y botones de acción.

  • La página principal de la aplicación muestra el título centrado y opcionalmente, solo botones para navegación y acceso al perfil, mostrando el avatar del usuario.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado center-aligned-top-app-bar y su comportamiento lo definimos con la clase CenterAlignedTopAppBar.

  • Para el título se usa un elemento de tipo h1 dentro de la barra de aplicación.

  • El botón de navegación opcional, debe llevar al atributo slot="navigation".

  • El botón de perfil opcional, debe llevar al atributo slot="avatar".

  • Para añadir elementos debajo del título, usa el atributo slot="adicional".

F. Vista secundaria

  • Cuando la barra de aplicación no está en la página principal, puede mostrar hasta 3 botones de ícono a la derecha. Deben ser las acciones más usadas.

  • Cuando hay más de 3 acciones, coloca las menos usadas en un menú que se despliega con un botón con el ícono ⁝ en Android y ⋯ en iOS.

  • Cuando en iOS no hay íconos de acción, el título debe ir centrado.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado top-app-bar y su comportamiento lo definimos con la clase TopAppBar.

  • Para el título se usa un elemento de tipo h1 dentro de la barra de aplicación.

  • El botón de navegación opcional, debe llevar al atributo slot="navigation".

  • Los botones de acción deben llevar al atributo slot="action".

  • El botón que abre y cierra el menú debe llevar al atributo slot="toggle".

  • El menú desplegable debe llevar al atributo slot="menu".

  • Para añadir elementos debajo del título, usa el atributo slot="adicional".

G. Campos de texto

  • Se usa una etiqueta flotante que se muestra grande cuando el contenido de campo es una cadena vacía y se muestra pequeña cuando el elemento se está capturando o cuando su contenido no está vacío.

  • Para que el contenido se muestra adecuadamente, se define el elemento personalizado text-field y su comportamiento lo definimos con la clase TextField.

  • Las etiquetas deben llevar al atributo slot="label".

  • Los campos de captura deben llevar al atributo slot="target".

  • Los textos de ayuda deben llevar al atributo slot="supporting".
